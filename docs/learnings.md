## Learnings

- **index subcommand dispatch must precede pflag parsing**: `Run()` checks `args[0] == "index"` before calling `pflag.Parse`. The index subcommand uses its own fresh `pflag.FlagSet`; if dispatch happened after main flag parsing, `--format` and `--wordlist` would be unknown flags to the main set.
- **pflag and io imports in run.go**: The main `run.go` didn't originally import `pflag` or `io` (those were in `config.go`). Adding `runIndex` to `run.go` requires adding both imports to `run.go` explicitly.
- **WriteIndexFile sorts entries in memory**: The index file format requires entries sorted by (system, value). WriteIndexFile collects all (system, value, word) triples across all 4 systems and sorts them in memory before writing. This is fine for typical word list sizes.
- **ParseWordListSlice refactors ParseWordList**: ParseWordList was refactored to call ParseWordListSlice internally, avoiding code duplication. The slice variant is needed by the index subcommand to iterate all words for bulk computation.
- **Backend auto-selection not yet wired**: `runFind` still uses `ParseWordList` (in-memory) regardless of file extension. Opening a .db or .idx file with `--find` will silently read it as a plain text file and find no valid Hebrew words. Backend auto-selection is tracked in wor-22g6.

- **Makefile CMD_DIR was wrong**: `./cmd/server` should be `./cmd/gematria`. Fixed before any other work.
- **staticcheck SA6003**: Range over a string directly (`for _, r := range "abc"`) instead of converting to `[]rune` first. golangci-lint catches this and fails the build.
- **Spec naming conflict Рђћ Letter function vs Letter type**: The spec defines `Letter(r rune) (Letter, error)` but Go does not allow a function and a type with the same identifier in the same package. The function was renamed to `LookupLetter(r rune) (Letter, error)` in `src/gematria.go`. Downstream code (Parse, Compute, CLI) must use `LookupLetter` instead of `Letter`.
- **Atbash Compute Рђћ do NOT call AtbashSubstitute**: The `systemValues[Atbash]` table already maps each original Hebrew rune to the Hechrachi value of its Atbash mirror. Calling `AtbashSubstitute` before lookup would use the substituted rune as a key in the atbash table, giving the wrong value (e.g., `atbash['Оф']` = 1 instead of 400 for aleph). `AtbashSubstitute` is for display-only use in output formatters.
- **Multi-word Latin aliases not supported in Parse**: Aliases like "kaf sofit" contain spaces. The `parsePureLatin` path uses `strings.Fields` (space-split), so "kaf sofit" becomes two tokens ["kaf", "sofit"]. This is intentional Рђћ users wanting sofit forms via Latin input should use single-word aliases or the Hebrew character directly.
- **staticcheck ST1005 Рђћ CLI error strings must be lowercase**: Even for user-facing CLI validation errors like `invalid value %q for --mispar`, the Go error string must be lowercase to satisfy staticcheck. The CLI output layer (run.go) adds the `Error:` prefix with capitalization when rendering to stderr.
- **ANSI constants must be used or unused linter fires**: When adding ANSI constants and color helpers to the CLI package, they must be used by at least one function in the same package or golangci-lint's `unused` check fails. Solution: define them in `output.go` (not `color.go`) so formatter methods can reference them. The `FormatError` implementations in `lineFormatter`/`cardFormatter` are a natural first user Рђћ they apply `ansiBold` wrapping when `useColor` is true. `jsonFormatter.FormatError` is a stub and does not use color.
- **errcheck in test Cleanup**: `t.Cleanup(func() { r.Close() })` is flagged by errcheck. Use `_ = r.Close()` to make the discard explicit and satisfy the linter.
- **GEMATRIA_MISPAR is lazy, GEMATRIA_OUTPUT is eager**: In `parseConfig`, `GEMATRIA_OUTPUT` is validated immediately because the output format controls how errors themselves are rendered. `GEMATRIA_MISPAR` is stored as-is without validation Рђћ the root package's `Compute()` handles validation via `validateSystem()`. Use `fs.Changed("mispar")` to detect whether the flag was explicitly set vs. coming from env.
- **staticcheck QF1012 Рђћ use fmt.Fprintf not WriteString(fmt.Sprintf)**: When building a `strings.Builder` with formatted output, use `fmt.Fprintf(&sb, ...)` directly rather than `sb.WriteString(fmt.Sprintf(...))`. golangci-lint flags the latter.
- **NewFormatter takes showAtbash**: The `--atbash` display flag must be threaded into `NewFormatter(output, useColor, showAtbash bool)`. The `wor-by4k` orchestration ticket must pass `cfg.Atbash` as the third argument when calling `NewFormatter`.
- **JSON nil slice Рєњ [] not null**: For JSON error responses, initialize `Suggestions: []string{}` (not nil) so it serialises as `[]` rather than `null`. Use `if une.Suggestions != nil` guard before assigning.
- **errcheck on fmt.Fprint to os.File**: `fmt.Fprint(w, s)` to an `*os.File` is flagged by errcheck. Use `_, _ = fmt.Fprint(w, s)` to satisfy the linter when discarding the return values is intentional.
- **strings.SplitN vs strings.Split for TSV parsing**: Using `strings.SplitN(line, "\t", 3)` causes the third field to absorb any extra tab-delimited columns (e.g., "shalom\tpeace\textra" Рєњ Meaning = "peace\textra"). Use `strings.Split` (no limit) and only consume `parts[0..2]`; extra columns are naturally ignored without any special-case code.
- **Go struct tag-ignoring conversion (S1016)**: When two struct types have identical field names and types but different struct tags (e.g., `Word` vs `remoteWordJSON`), Go allows direct type conversion `Word(w)` since Go 1.8 ignores struct tags for conversion purposes. staticcheck S1016 flags manual struct-literal copies when a type conversion suffices.
- **batchLineError wrapper for per-line error context**: processBatch wraps compute errors with a `batchLineError{cause, lineNum}` before passing to `formatter.FormatError`. Plain-text formatters get "line N: msg" automatically via `err.Error()`. The JSON formatter must explicitly check `errors.As(err, &ble)` to extract `lineNum` and put it in a dedicated `"line"` JSON field; use the inner error's message for the `"error"` field rather than the wrapper's message.
- **NewFormatterWithLookup for card/json lookup context**: `NewFormatter` kept its 3-param signature (delegates to `NewFormatterWithLookup` with 0/"" defaults). Card and JSON formatters need the lookup target value and system to produce headers/JSON fields Рђћ use `NewFormatterWithLookup(output, useColor, showAtbash, findValue, findSystem)` when in the --find path. The `--find` orchestration in run.go (wor-tyjq) should call `NewFormatterWithLookup` with `cfg.FindValue` and `cfg.Mispar`.

- **openWordSource returns (WordSource, io.Closer, error) Рђћ not type assertion at call site**: Returning a distinct io.Closer (nil when no cleanup needed) is cleaner than a type assertion in runFind. SQLite returns `src.(io.Closer)`; the index backend returns the `*os.File` as closer (the indexWordSource itself does not implement io.Closer); remote returns nil; memory opens, parses, and closes the file immediately then also returns nil.
- **runFind must receive getenv to thread GEMATRIA_WORDLIST_TOKEN into openWordSource**: Adding a new env var consumed only inside runFind means threading getenv down from Run() to runFind(). The function signature becomes `runFind(cfg, formatter, stdout, stderr, getenv)`.
- **httptest.NewServer is the right tool for remote backend CLI tests**: Real HTTP round-trips via httptest avoid mocking gematria internals while keeping tests hermetic. The server URL (e.g. `http://127.0.0.1:PORT`) satisfies the "http://" prefix check in openWordSource automatically.
- **Oracle tests must sort results before comparing**: SQLite and Index backends may return results in slightly different order than in-memory ParseWordList. Sort both result slices by Hebrew text before asserting equality to avoid false failures from ordering differences.
- **bytes.NewReader vs strings.NewReader for NewIndexWordSource tests**: NewIndexWordSource takes an io.ReadSeeker. strings.NewReader implements io.ReadSeeker; bytes.NewReader also does. When building an index file in-memory for tests, write to a bytes.Buffer then construct a bytes.NewReader from buf.Bytes() to get an io.ReadSeeker without needing a temp file.

- **sofitMap defined in transliteration_academic.go, shared with israeli**: Both transliteration schemes apply the same sofit substitution (ОЏРєњОџ, ОъРєњОЮ, ОаРєњОЪ, ОцРєњОБ, ОдРєњОЦ). sofitMap is defined once in transliteration_academic.go and referenced by transliteration_israeli.go Рђћ it is not duplicated or re-defined.
- **israeliVowels must be used or golangci-lint unused fires**: Define israeliVowels as a map[byte]bool and reference it inside israeliHasConsonantAfter() (the lookahead helper) to skip over vowels explicitly. This documents intent and satisfies the linter without adding functional complexity.
- **israeliHasConsonantAfter lookahead for word-final detection**: To determine if a vowel (a/e) is word-final, scan ahead for any consonant (single or multi-char). Return on first consonant found; no need to track proper advancement since we return immediately. Vowels and unknown chars just advance by 1.
- **consonant tables are shared between academic and israeli**: academicMultiChar and academicSingleChar are referenced directly in transliteration_israeli.go Рђћ no need to duplicate. The spec explicitly states the consonant tables are identical for both schemes.
- **transliterateAcademic handles single word-parts (no spaces)**: The internal scheme function receives one space-free word-part. Space-splitting (for quoted compound inputs like "shalom emet") is the responsibility of the public Transliterate() function (gc-d6hf). Do not add space-splitting logic inside scheme-specific functions.
- **apostrophe `'` as Aleph in academic scheme**: The single-char map uses `academicSingleChar['\''] = 'Ољ'`. The byte value of apostrophe in a Go map literal needs to be escaped as `'\''`. This is fine as a raw byte key Рђћ it does not conflict with the multi-char table which only has letter pairs.
- **Transliterate splits on spaces; Hebrew parts bypass scheme functions**: The public Transliterate() function calls strings.Fields, then for each word-part uses isAllHebrew() to decide: all-Hebrew parts go directly through LookupLetter; Latin parts dispatch to the per-scheme transliterator. This is necessary because transliterateAcademic/transliterateIsraeli operate byte-by-byte on ASCII and return an error for Hebrew Unicode characters (multi-byte UTF-8).
