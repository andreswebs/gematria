# Implement pre-computed index file WordSource backend

Create src/backend_index.go in the root package. Implement indexWordSource, an unexported struct that satisfies both WordSource and io.Closer by reading a sorted TSV index file generated by gematria index --format index (index-subcommand task). Provides O(1) lookup after a single O(n) scan on open.

## Implementation Details

### Index File Format

Sorted TSV, one line per word-system pair:
  <system>\t<value>\t<hebrew>\t<transliteration>\t<meaning>

Lines are sorted by system (lexicographic) then by value (numeric ascending). Blank lines and lines beginning with '#' are treated as comments and skipped. The first line must be the magic comment:
  # gematria-index v1

### In-Memory Jump Table (built on Open)

  type indexRange struct {
    offset int64 // byte offset of first matching line in file
    count  int   // number of consecutive lines with this (system, value)
  }

On OpenIndexWordSource, scan the file sequentially once:
- For each data line, record current byte offset before reading it
- Parse (system, value) from the first two fields
- Group consecutive identical (system, value) pairs into an indexRange
- Store in map[System]map[int]indexRange

The jump table holds only byte offsets and counts — not word data — so memory cost is proportional to the number of distinct (system, value) pairs, not the number of words.

### Exported Constructor

  // OpenIndexWordSource opens a pre-computed gematria index file.
  // The file must have been produced by "gematria index --format index".
  // Callers should check whether the returned WordSource also implements
  // io.Closer and call Close when done.
  func OpenIndexWordSource(path string) (WordSource, error)

- Opens the file with os.Open (OK — only CLI layer calls this; wait, root package cannot import os)

IMPORTANT CONSTRAINT: Root package must not import "os". Therefore OpenIndexWordSource accepts an io.ReadSeeker, not a path string. The CLI layer (backend-selection task) is responsible for opening the file with os.Open and passing the *os.File (which implements io.ReadSeeker) to the constructor.

Revised signature:
  func NewIndexWordSource(r io.ReadSeeker) (WordSource, error)

The CLI layer type-asserts the returned WordSource to io.Closer for cleanup; since *os.File implements io.Closer, the CLI layer closes the file directly:
  f, err := os.Open(path)
  ...
  src, err := gematria.NewIndexWordSource(f)
  defer f.Close()

indexWordSource stores the io.ReadSeeker but need not implement io.Closer itself (file lifecycle managed by caller). Remove the io.Closer compile-time check for this backend.

Compile-time check:
  var _ WordSource = (*indexWordSource)(nil)

### FindByValue

1. Look up indexRange in map[system][value]. If absent, return (nil, false, nil).
2. Seek to range.offset in the underlying io.ReadSeeker.
3. Read range.count+1 lines sequentially. Parse the hebrew, transliteration, meaning fields from columns 2, 3, 4 (0-indexed).
4. Return results[:min(len,limit)], len(results)>limit, nil.

Word parsing: split on '\t', expect 5 fields minimum. Fields 3 and 4 may be absent (treated as empty string).
